Summary

An external IP (218.92.0.56) launched a brute force attack against the RDP service (port 3389) of host Matthew (172.16.17.148). After 14 consecutive failed login attempts across multiple usernames, the attacker successfully authenticated as user matthew. The firewall permitted all traffic throughout. The host was immediately isolated via EDR upon confirmation of unauthorized access.

Investigation Walkthrough

1. Initial Alert Review

Documented all key metadata: hostname (Matthew), source IP (218.92.0.56), destination IP (172.16.17.148), protocol (RDP/3389), and timestamp. External RDP traffic immediately raised suspicion of a brute force attempt.

2. IP Reputation — All Platforms Confirmed Malicious

Checked 218.92.0.56 on VirusTotal, AbuseIPDB, and LetsDefend TI. All three returned malicious verdicts, confirming the IP as a known attack source with no ambiguity.

3. Authentication Log Analysis

Filtered logs by source and destination IP. Identified a clear brute force pattern targeting multiple usernames: matthew, sysadmin, admin, guest.

Event IDEvent TypeCount

4. Scope Assessment

Checked if other hosts were targeted by the same IP. Log analysis confirmed only host Matthew was affected — targeted attack, not a broad scan.

5. Successful Login Confirmed

Event ID 4624 confirmed a successful logon for user matthew from the external IP. The firewall was marked Allowed throughout — no network controls blocked the attack.

6. Containment

Immediately isolated host Matthew via EDR to terminate the attacker's session, prevent lateral movement, and stop post-exploitation activity.

Final Classification
True Positive — Confirmed Unauthorized Access
Recommendations
  • Immediately reset password for user matthew
  • Block source IP 218.92.0.56 at the firewall and perimeter
  • Disable public-facing RDP — use a VPN or jump server instead
  • Enable Multi-Factor Authentication (MFA) for all RDP access
  • Implement account lockout after 5 failed attempts (Event ID 4625)
  • Monitor for Event ID 4625 spikes as an early brute force detection signal
Key Learnings
  • Brute force attacks are patient. 14 attempts before success shows that weak or common passwords will eventually be guessed given enough time.
  • Windows Event IDs 4624/4625 are fundamental. Knowing these by heart is essential for detecting RDP attacks quickly in any SOC role.
  • Exposed RDP is a critical risk. Allowing RDP from the internet without MFA or lockout policies is an easily exploitable attack surface.
  • Multi-source TI builds confidence. All three platforms confirming the IP removed all doubt and accelerated the triage decision.
  • Speed of containment matters. Every minute after a confirmed successful login increases lateral movement risk.
