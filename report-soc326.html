Summary

An alert triggered by a suspicious MX record change on the typosquatted domain letsdefwnd[.]io — impersonating letsdefend.io. A phishing email from voucher@letsdefwnd[.]io was delivered and clicked by user [email protected]. Log analysis confirmed active C2 communication with IP 45.33.23.183 (84 AbuseIPDB reports). Despite the URL appearing clean on initial checks, EDR confirmed browser activity to the fake domain. Email deleted and host contained.

Investigation Walkthrough

1. Initial Alert Review

Noted the suspicious domain letsdefwnd[.]io — immediately recognizable as a typosquatted version of letsdefend.io with a single character swap ('e' → 'w') designed to deceive users at a glance.

2. IP Reputation Analysis

All associated IPs flagged as malicious on VirusTotal. LetsDefend TI returned no entries — highlighting that no single platform catches everything and multiple sources are always necessary.

3. Log Analysis — Active Communication Confirmed

Filtered logs by each malicious IP. Confirmed that 45.33.23.183 successfully communicated with host 172.16.17.162 — this was the pivotal finding that escalated the alert from suspicious to confirmed active threat.

4. URL Reputation — Clean Result Was Misleading

Checked the URL on VirusTotal, Cisco Talos, and LetsDefend TI — all three returned clean results. However, the alert was not closed. Newly registered phishing domains often evade reputation tools. The associated IPs were already confirmed malicious — investigation continued.

5. AbuseIPDB — 84 Reports Confirm Malicious

Checking 45.33.23.183 on AbuseIPDB returned 84 community abuse reports, firmly confirming the IP as malicious and overriding the earlier clean URL verdict. This is a key example of why AbuseIPDB fills gaps other platforms miss.

6. EDR Analysis

EDR confirmed network connections to 45.33.23.183 and browser activity (Chrome) accessing letsdefwnd[.]io — proving the user clicked the phishing link and directly exposed the host to attacker infrastructure.

7. Containment

Deleted the phishing email from the email security system. Isolated host 172.16.17.162 via EDR to stop communication with the malicious domain and IP.

Recommendations
  • Block IP 45.33.23.183 at the firewall
  • Block domain letsdefwnd[.]io in email gateway and DNS filter
  • Remove all emails from voucher@letsdefwnd[.]io across all mailboxes
  • Reset credentials for [email protected] in case of credential harvesting
  • Implement DNS filtering to catch single-character domain variations of trusted domains
  • Conduct phishing awareness training on domain spoofing and typosquatting
Key Learnings
  • A clean URL verdict is not the final word. Three platforms returned clean for the URL — AbuseIPDB and EDR told a different story. Always correlate across multiple sources.
  • Typosquatting is subtle but detectable. A single character difference is enough. Inspecting domain names carefully is an essential SOC analyst habit.
  • MX record changes on lookalike domains are a red flag. When a domain resembling a trusted brand configures mail exchange records, a phishing campaign is almost always imminent.
  • AbuseIPDB fills gaps that VirusTotal misses. Different TI platforms index different data — 84 reports on AbuseIPDB vs. clean on VirusTotal is a clear example of why using both matters.
  • Correlating email, logs, and EDR confirms the full picture. The email showed delivery, logs confirmed IP communication, EDR confirmed the user clicked — no single source was conclusive alone.
